
# 类与对象
这几天学了类与对象，现在来复习一下
## 类
我认为就是功能超级拓展的结构体
### 定义类
```java
class Cat{
	String name;
	int age;
	public int printname(){
		System.out.println(name);
	}
	public Cat(String name,int age){
		this.name=name;
		this.age=age;
	}
}
```
上述就是一个对类的定义其中Cat就是类名,name,age就是属性或者称为字段，printname()就是成员方法，Cat就是构造器，之后会一一细说
创建完类之后如果想要使用就需要定义对象，通过对象来使用类，这一步放在main中，方法如下：
```java
public static void main(String[] args){
	Cat c=new Cat();
}
```
实际上，对象的开辟有一定的顺序和方法，这点笔记里有，这里也不方便写，就看笔记吧
#### 属性
其实就是类中的全局变量，作用域是整个类，包括基本数据类型和引用数据类型，因此可以包括数组和对象，在这里定义时如果没有初始化系统会定义默认值，0，null，false，0.0，并且还可以被其他类使用和被修饰词修饰。这里要和方法中定义的局部变量相区分
#### 成员方法
在我看来，其实就是c中的函数，并且定义方式也很像
```java
【访问限制符】【返回类型】【方法名】(形参列表){
	【代码块】；
	return ---;//返回为void无return
}
```
其中访问限制符包括 public protected 默认,private，这个在下一章会详细讲。而返回值包括基本数据类型、void和数组。	
- 接下来说一下方法的细节：
1. return时可以返回变量，常量和表达式。
2. 方法名书写时用驼峰命名法：第一个单词首字母小写，其余大写。
3. 方法不能嵌套定义
4. 同一个类中的方法调用时可以直接调用，不用创建对象
5. 当向方法中传入对象时，编译器也会开辟一个栈，并指向这个对象的堆区，因此，通过方法改变对象的值会使得整个对象的值改变
6. 方法中的变量是局部变量，这里要注意几点：1. 在方法中定义的局部变量一定要初始化。2. 局部变量和全局变量可以重名，但访问时遵从就近原则，即优先调用方法内的局部变量，而局部变量不能重名。3.与全局变量不同，局部变量只有本类的对应方法能用，且不能加修饰词
7. 在main中，无论是创建类对象还是使用方法，后面都有()，即便里面没有参数如：
```java
public static void main(String[] args){
Cat c=new Cat();
c.printname(); 
```
- 递归
现在我对他的理解就是先递，当达到退出递归条件时就会归，多的不想说，练的还是少了
- 重载
Java中同一个类可以有多个同名的方法的存在，但每个方法的参数列表都要不同，这里的不同可以是类型不同，数量不同，顺序不同。
如：
``` java
public double add(int x,int y){}
public double add(double x,int y){}
public double add(int x,double y){}
public double add(int x,int y,int z){}
```
这三个就是重载，注意：形参的名字无影响
- 可变参数
在Java中，对于功能相同，参数类型相同，但参数数量不同的方法可以使用可变参数
如：
```java
public void findScore(int ... scores){...}
```
这里要注意一些细节：
8. 参数个数可以为零。
9. 可变参数可以与普通类型参数（基本数据类型和引用数据类型）放在一起，但普通类型参数要在可变参数前
如：
```java 
public void findScores（String name,int ... scores){...}
```
#### 构造器
在传统的类中，创建对象之后要对各个属性赋值，当创建同类的多个对象时，就会很麻烦，于是Java提供了一种快速完成对新对象的初始化的工具--构造器
它的基本语法为：
```java
class Cat1{
	String name;
	int age;
	public Cat1(String n,int a){
		name=n;//注意此处是将传进来的值赋给全局变量
		age=a;
	}
	【修饰符】【构造器名】（形参列表）{
		执行语句；
	}
}
```
public Cat1就是一个构造器，后面的时构造器的抽象结构。
接下来是一些构造器的细节：
1. 构造器没有返回值，连void都没有，这是区分构造器的一个好方法
2. 构造器名和类名相同
3. 一个类可以有多个不同的构造器，但形参列表不能相同，即构造器重载
4. 构造器不是方法，不能通过对象使用
5. 其实如果没有定义构造器，系统也会自动定义一个无参构造器，即：没有形参，没有执行语句，但如果定义了构造器，这个无参构造器就会被覆盖，如果想要，就要显式定义一下，即Cat1(){}。这一点在之后的继承很重要。
6. 使用构造器时只是初始化变量，还没有开辟内存，如果要开辟内存，要用new，开辟之后，只能使用一次，如果要多次使用，就要用对象变量
如：
```java
public static void main(String[] args){
	new Cat1("小花",2);//只是创建了对象，但没有对象变量
	Cat1 c=new Cat1("小花",2);//有对象变量
}
```
#### this
类的功能这么多，这么繁杂，如何区分？Java中提供了this语句，当创建一个类时，就会隐式创建一个this指向这个类，用来区分本类的属性，方法和构造器。
如上面Cat1的构造器就可以写为：
```java
public Cat1(String name,int age){
	this.name=name;
	this.age=age;
}
```
当方法传输是，也可以通过this实现形参与属性的名字相同。
此外，在构造器中，this还可以实现在构造器中访问另一个构造器，但注意，这种用法在一个构造器中只能使用一次，并且要在被访问的构造器前面。
